# TradingAgents-Astock：面向 A 股市场的多 Agent 投研框架

## 一句话介绍

TradingAgents-Astock 是一套面向中国 A 股市场的开源多 Agent 投研框架。它将大型语言模型与结构化金融数据、新闻信息、资金流和政策信息相结合，通过多个分析角色协同工作，生成更接近真实投研流程的研究报告与交易建议。

## 为什么它值得被关注

A 股市场具有独特的复杂性：政策驱动、资金博弈、限售解禁、板块轮动与消息情绪交织。传统单模型或单角色分析工具，往往难以同时覆盖这些维度。TradingAgents-Astock 通过“多角色协作 + 结构化数据 + 规则约束”的方式，试图将投研过程从单一判断，升级为更接近专业研究团队的协同分析。

## 项目核心价值

### 1. 面向 A 股深度适配，而非简单移植

项目基于 TradingAgents 的多 Agent 思路进行深度改造，不是把美股框架简单翻译为 A 股。它在以下层面做了针对性优化：

- 数据层对接 A 股真实场景：mootdx、腾讯财经、东方财富、新浪财经、同花顺、财联社等来源
- 角色设计扩展为 7 位 Analyst：覆盖市场、情绪、新闻、基本面、政策、游资与解禁等维度
- 交易规则与 A 股约束对齐：T+1、涨跌停、手数、交易时段等因素纳入分析链路
- 输出更符合中文市场语境的研究报告与决策表达

### 2. 多 Agent 协同，提升研究深度与解释性

系统不是单一模型给出结论，而是将分析任务拆分为多个专业角色：

- 市场分析师：关注 K 线、量价、技术指标
- 舆情分析师：分析市场情绪与讨论热度
- 新闻分析师：整合消息、公告与宏观事件
- 基本面分析师：研究财报、估值与盈利能力
- 政策分析师：把政策变化、产业导向纳入判断
- 游资追踪师：关注主力资金、龙虎榜与资金异动
- 解禁监控师：关注限售股解禁与供给压力

这些角色之间通过 Bull/Bear 辩论、风险辩论以及研究经理与组合经理的汇总决策，形成更完整的投研闭环。

### 3. 开箱即用，降低使用门槛

项目提供了 CLI 与 Streamlit Web UI 两种使用方式，适合不同类型用户：

- 开发者：可通过 Python 接口快速集成到自己的研究流程
- 研究人员：可直接通过命令行运行分析任务
- 普通用户：可通过 Web 页面输入股票代码，快速生成分析结果

此外，项目致力于减少部署复杂度，支持通过 pip 安装直接使用，并且提供 Docker 部署能力。

## 产品亮点

### 低门槛接入

- Python 3.10+
- 通过 pip 安装即可使用
- 支持多种主流大模型提供商：MiniMax、DeepSeek、Qwen、GLM、OpenAI、Anthropic、Google Gemini、Ollama 等

### 数据链路完整

- 无需依赖 Tushare 等积分墙服务
- 采用直连 HTTP / TCP 数据源获取行情、财务、新闻与资金流信息
- 针对东方财富接口做了节流与限速设计，降低被封禁风险

### 可视化与交互体验

- 内置 Streamlit Web UI
- 支持在界面中选择模型、配置环境变量并提交分析任务
- 支持结果展示、历史记录和报告导出能力

## 适用场景

### 1. 投研辅助

适合研究员、量化从业者和交易团队快速搭建 A 股研究辅助流程，用于分析消息面、资金面、基本面与政策面的综合影响。

### 2. 教学与研究

对于高校、培训机构和技术研究团队，项目可作为“多 Agent 金融分析系统”的实践样例，帮助理解 LLM 在金融场景中的应用方式。

### 3. 开源社区与技术试验

项目以 Apache 2.0 开源协议发布，便于二次开发、实验和改造，适合社区成员参与迭代和扩展。

## 与传统工具的区别

相比传统的单模型问答式分析工具，TradingAgents-Astock 的优势在于：

- 不仅给出答案，更提供分析过程的角色分工与辩论结构
- 更强调 A 股市场特有信息源与制度约束
- 将“模型推理”与“市场信息结构化获取”结合起来
- 更适合研究型、实验型和教育型应用场景

## 项目定位总结

TradingAgents-Astock 不是单纯的“股票聊天机器人”，而是一套面向 A 股的研究型 Agent 框架。它希望把 LLM 的推理能力与金融研究流程相结合，为用户提供一个更接近专业投研团队协作方式的分析体验。
